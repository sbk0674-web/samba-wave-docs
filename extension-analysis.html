왜 확장앱을 쓸까?

무재고 위탁판매 솔루션이 브라우저 확장앱을 사용하는 이유와
크롤링 대비 확장앱 방식의 기술적 장점을 분석합니다

CHAPTER 01

먼저 쉬운 비유로 이해하기

인터넷 쇼핑몰에서 상품 정보를 가져오는 걸, 도서관에서 책 내용을 베끼는 것에 비유해볼게요.

🤖

확장앱 없이 (서버가 직접)

로봇을 도서관에 보내서 책을 읽게 하는 것과 같아요. 그런데 도서관 입구에서 "로봇은 출입금지!"라는 표지판을 보고 쫓겨납니다. 도서관 회원증(로그인)도 없고, 사서가 로봇을 의심해서 책을 안 보여줘요.

👤

확장앱으로 (사람이 직접)

내가 직접 도서관에 가서 회원증을 보여주고, 책을 펼쳐 읽으면서 중요한 내용만 메모하는 거예요. 사서는 아무런 의심 없이 책을 보여주고, 나는 편하게 메모를 가져옵니다.

👓

확장앱 = "특수 안경"

브라우저 확장앱은 특수 안경 같은 거예요. 내가 평소처럼 쇼핑몰을 구경하면, 안경이 자동으로 상품 정보를 읽어서 메모장(서버)에 적어줘요. 쇼핑몰 입장에서는 그냥 평범한 손님이 구경하는 것처럼 보입니다.

📋

왜 둘 다 확장앱을 쓸까?

쇼핑몰들은 로봇(서버)이 데이터를 긁어가는 걸 적극적으로 차단해요. CAPTCHA, 로그인 강제, IP 차단 등... 하지만 실제 사람(브라우저 확장앱)은 차단할 수 없어요. 그래서 더망고와 오버링크 모두 확장앱을 사용합니다.

CHAPTER 02

확장앱을 쓰는 진짜 이유 5가지

더망고와 오버링크 모두 확장앱을 사용하는 건 같습니다. 서버가 직접 사이트에 접근하면 생기는 문제들을 확장앱이 해결해주기 때문이에요.

문제 서버가 직접 접근하면 확장앱을 쓰면
봇 감지 "로봇이 접속 중입니다" 차단 차단됨 실제 사람이 보는 브라우저 통과
로그인 CAPTCHA, 2단계 인증 등 자동 로그인 거의 불가능 이미 로그인된 브라우저 세션 그대로 사용
쿠키/세션 서버가 직접 관리해야 함 (만료, 갱신, 암호화 복호화...) 브라우저가 알아서 관리해줌
자바스크립트 SPA 사이트는 Puppeteer로 무거운 브라우저를 돌려야 함 브라우저가 이미 다 실행해놓은 완성된 화면을 읽음
개발 난이도 사이트마다 1~2주 개발, 봇 우회 기법 연구 매우 어려움 개발자도구에서 셀렉터만 찾으면 끝 1~2시간

왜 이렇게 차이가 날까?

확장앱은 "이미 완성된 화면을 보고 메모"하는 거예요. 개발자도구(F12)에서 원하는 부분의 이름(셀렉터)만 찾으면 끝이에요.

서버에서 직접 수집하면 "빈 종이부터 그림을 다시 그려야" 해요. 봇 감지 우회, 로그인 자동화, 쿠키 암호화 복호화, JavaScript 실행 환경 구축... 같은 데이터를 얻는데 10~50배 더 어렵습니다.

SUMMARY

정리

확장앱을 쓰는 이유

  • 쇼핑몰의 봇 차단을 피할 수 있다 (실제 사람의 브라우저이니까)
  • 이미 로그인된 상태를 그대로 쓸 수 있다 (로그인 자동화 불필요)
  • 브라우저가 다 그려놓은 완성된 화면을 읽으면 된다 (JS 렌더링 불필요)
  • 개발이 10배 이상 쉽고, 유지보수도 간단하다
