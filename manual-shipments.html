1. 전송 대상 선택

상품을 전송하기 전에 전송할 마켓 계정과 카테고리를 선택합니다.

지원 마켓 (17개)

  • 국내 마켓 — 스마트스토어, 쿠팡, 11번가, 롯데ON, SSG, GS샵, G마켓, 옥션, 카페24, 롯데홈쇼핑, KREAM
  • 해외 마켓 — Amazon, Rakuten, Buyma
  • 기타 — Toss
ESMPlus 통합 전송 G마켓과 옥션은 ESMPlus API를 통해 동시에 전송됩니다. G마켓을 선택하면 옥션에도 자동으로 함께 전송됩니다.

전송 전 체크리스트

  • 마켓 스토어 연결이 완료되었는지 확인 (설정 > 마켓 연결)
  • 카테고리 매핑이 완료되었는지 확인
  • 가격 정책이 적용되었는지 확인
  • 소싱처 원가가 최신 상태인지 확인
  • 등록 상품명이 적절한지 확인
  • 상품 이미지가 정상적인지 확인

계정 선택

  1. 상품관리 페이지에서 전송할 상품을 선택합니다.
  2. 상품전송 버튼을 클릭합니다.
  3. 전송 대상 마켓을 선택합니다. (스마트스토어, 쿠팡, 11번가 등)
  4. 해당 마켓에 연결된 계정을 선택합니다.

2. 상품 전송 실행

선택한 상품을 마켓에 등록합니다.

  1. 전송 대상 마켓과 계정을 선택한 후 전송 시작을 클릭합니다.
  2. 전송이 시작되면 진행 상태가 표시됩니다.
  3. 각 상품별로 전송 성공/실패 결과가 실시간으로 업데이트됩니다.
  4. 전송이 완료되면 요약 결과가 표시됩니다.

3. 전송 로그 확인

상품 전송 이력과 결과를 확인합니다.

로그 확인 방법

  • 전송일시 — 전송이 실행된 날짜와 시간
  • 마켓 — 전송 대상 마켓명
  • 전송 건수 — 총 전송 시도 상품 수
  • 성공/실패 — 성공 및 실패 건수
  • 에러 메시지 — 실패한 경우 에러 상세 내용

4. 마켓삭제 / 판매중지

마켓에 등록된 상품을 삭제하거나 판매를 중지합니다.

절대 혼동 금지 — 마켓삭제 ≠ 판매중지 마켓삭제와 판매중지(SUSPENSION)는 완전히 다른 동작입니다. 혼동하면 복구 불가능한 상황이 발생합니다.

마켓삭제 — 마켓 API를 통해 상품을 완전히 삭제합니다. 삭제된 상품은 복구 불가능합니다. 신중하게 사용하세요.
판매중지 (SUSPENSION/STOP) — 판매를 일시 중단합니다. 상품은 마켓에 유지되며, 언제든 재전송하여 판매 재개가 가능합니다.
강제삭제 — DB에서만 전송 기록을 삭제합니다. 마켓에서 이미 삭제된 경우에 사용합니다.

마켓삭제

마켓 API를 통해 해당 마켓에서 상품을 완전히 삭제합니다. 삭제 후 복구가 불가능하므로 반드시 신중하게 사용하세요.

  1. 삭제할 상품을 선택합니다.
  2. 마켓삭제 버튼을 클릭합니다.
  3. 삭제할 마켓을 선택합니다.
  4. 확인 모달에서 삭제 실행을 클릭합니다.

판매중지 (SUSPENSION)

상품을 삭제하지 않고 판매 상태만 중지로 변경합니다. 상품은 마켓에 그대로 유지되며, 나중에 다시 전송하여 판매를 재개할 수 있습니다.

강제삭제

마켓 API를 호출하지 않고 삼바웨이브 DB에서만 전송 기록을 삭제합니다. 마켓에서 이미 수동으로 삭제한 경우, 또는 마켓 연동이 불가한 경우에만 사용합니다.

주의 강제삭제는 삼바웨이브의 전송 기록만 삭제합니다. 마켓에 등록된 실제 상품은 삭제되지 않으므로, 마켓에서 수동으로 삭제해야 합니다.

5. 오토튠 (자동 가격/재고 업데이트)

마켓에 등록된 상품의 가격과 재고를 자동으로 최신 상태로 유지합니다.

오토튠 동작 방식

  • 소싱처 원가 변동 시 정책에 따라 마켓 판매가를 자동 재계산하여 업데이트
  • 소싱처 재고 변동 시 마켓 재고를 자동 동기화
  • 오토튠은 마켓에 등록된 상품에만 동작합니다 (미등록 상품 제외)

재고 업데이트 규칙

  • 실재고 (1개 초과) — 소싱처 재고 그대로 반영
  • 불명확 (1개) — 99로 설정 (재고 불확실 시 안전 수량 사용)
  • 품절 (0개) — 0으로 설정 (판매 중단)
TIP 정책에서 마진율을 변경하면 오토튠이 자동으로 해당 정책이 적용된 모든 마켓 상품의 판매가를 업데이트합니다.

6. 전송 오류 해결

자주 발생하는 전송 오류와 해결 방법입니다.

카테고리 미매핑

해당 상품의 소싱처 카테고리가 마켓 카테고리에 매핑되지 않았습니다.

해결: 카테고리매핑 페이지에서 해당 카테고리를 매핑합니다.

API 인증 오류

마켓 API 인증 정보가 만료되었거나 잘못되었습니다.

해결: 설정 페이지에서 해당 마켓의 API 키를 재발급받아 업데이트합니다.

필수 항목 누락

마켓에서 요구하는 필수 입력 항목(브랜드, 원산지 등)이 비어있습니다.

해결: 상품 편집에서 누락된 항목을 입력합니다.

이미지 오류

상품 이미지가 마켓의 규격(크기, 용량, 형식)에 맞지 않습니다.

해결: 이미지를 규격에 맞게 변환하거나 교체합니다.

TIP 전송 로그에서 에러 메시지를 클릭하면 상세 내용을 확인할 수 있습니다. 대부분의 오류는 데이터 보완으로 해결됩니다.
