1. 정책 생성

가격 정책은 소싱가(원가)에서 마켓 판매가를 자동 계산하기 위한 규칙입니다.

  1. 좌측 메뉴에서 정책관리를 클릭합니다.
  2. + 새 정책 버튼을 클릭합니다.
  3. 정책명을 입력합니다. (예: "기본정책", "고마진정책", "해외마켓정책")
  4. 공통 가격 설정을 입력합니다.
  5. 저장 버튼을 클릭합니다.

공통 가격 설정 항목

  • 마진율 (%) — 원가 대비 수익률. 예: 30%로 설정 시 원가의 1.3배
  • 배송비 (원) — 소싱처에서 받는 배송비. 판매가에 포함됩니다.
  • 추가비용 (원) — 포장비, 기타 비용 등 추가로 반영할 금액

2. 마켓별 정책

각 마켓의 특성에 맞게 개별 설정을 추가할 수 있습니다.

마켓별 설정 항목

  • 수수료 (%) — 마켓에서 부과하는 판매 수수료율. 마켓마다 다릅니다.
  • 배송비 (원) — 해당 마켓에서의 배송비 설정. 무료배송 여부 결정
  • 출고일 (일) — 상품 출고까지 소요 일수. 소싱처 발송 시간을 고려하여 설정
  • 재고 제한 (개) — 마켓에 등록할 재고 수량 제한

3. 정책 적용

생성한 정책을 수집 그룹 또는 개별 상품에 연결합니다.

그룹에 정책 연결

  1. 상품수집 페이지로 이동합니다.
  2. 좌측 트리에서 정책을 적용할 그룹을 선택합니다.
  3. 그룹 상세에서 정책 드롭다운을 클릭합니다.
  4. 적용할 정책을 선택합니다.
  5. 해당 그룹의 모든 상품에 정책이 일괄 적용됩니다.

개별 상품에 정책 적용

  1. 상품관리 페이지에서 상품을 선택합니다.
  2. 상단 메뉴의 정책 적용을 클릭합니다.
  3. 정책을 선택하면 선택된 상품에만 적용됩니다.
참고 그룹 정책과 개별 정책이 충돌할 경우, 개별 상품에 직접 적용한 정책이 우선됩니다.

4. 가격 계산 방식

정책이 적용된 상품의 판매가는 다음 공식으로 자동 계산됩니다.

원가(소싱처 최대혜택가) 기준

원가는 소싱처의 최대혜택가(쿠폰/포인트가 적용된 최저가)를 기준으로 합니다.

  • 무신사 최대혜택가: AG쿠폰은 제외, 일반 쿠폰 및 포인트만 반영
  • 소싱처별 최대혜택가는 주기적으로 자동 갱신됩니다.
판매가 = 원가(최대혜택가) x (1 + 마진율/100) + 배송비 + 추가비용

계산 예시

원가 50,000원, 마진율 30%, 배송비 3,000원, 추가비용 1,000원인 경우:

50,000 x 1.3 + 3,000 + 1,000 = 69,000원

마켓별 최종 판매가

마켓별 수수료가 추가로 반영되어 마켓마다 판매가가 달라질 수 있습니다.

마켓 판매가 = 판매가 / (1 - 수수료율/100)

예시: 판매가 69,000원, 스마트스토어 수수료 6%인 경우:

69,000 / (1 - 0.06) = 73,404원 (100원 단위 반올림)
TIP 가격 계산은 모두 자동으로 수행됩니다. 정책 설정만 정확히 입력하면 마켓별 판매가가 자동으로 결정됩니다.

5. 재고 업데이트 규칙

소싱처 재고 상태에 따라 마켓에 등록되는 재고 수량이 자동으로 결정됩니다.

  • 실재고 (1개 초과) — 소싱처 재고 그대로 마켓에 반영
  • 불명확 (1개) — 99로 설정 (재고를 특정할 수 없는 경우 안전 수량 사용)
  • 품절 (0개) — 0으로 설정 (마켓에서 품절 표시)
참고 오토튠이 활성화된 경우, 정책 변경 또는 소싱처 가격/재고 변동 시 마켓의 판매가와 재고가 자동으로 업데이트됩니다. 정책 수정 후 별도로 마켓에 반영할 필요가 없습니다.
오토튠 연동 정책에서 마진율, 배송비 등을 변경하면 오토튠이 자동으로 해당 정책이 적용된 모든 마켓 등록 상품의 판매가를 업데이트합니다.
