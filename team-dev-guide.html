삼바웨이브 팀 개발 가이드

10명이 동시에 코드를 수정해도 충돌 없이 개발하는 방법.
처음 개발하는 분도 이 가이드만 따라하면 됩니다.

1

우리 프로젝트 이해하기

코드를 쓰기 전에, 우리 프로젝트가 어떤 구조인지 먼저 이해합시다.

🧩 플러그인 = USB 장치

컴퓨터 본체(우리 시스템)는 이미 완성되어 있고, 마우스(쿠팡), 키보드(스마트스토어)처럼 USB만 꽂으면 바로 인식됩니다.
우리 플러그인도 마찬가지 — 파일 하나 만들면 자동으로 인식됩니다. 다른 파일을 수정할 필요가 없습니다.

🏪 판매처(마켓) 플러그인

쿠팡, 스마트스토어, 11번가 같은 곳에 상품을 등록하는 코드

역할: 상품 데이터를 해당 마켓 형식으로 변환하고 전송

📦 소싱처 플러그인

무신사, 올리브영 같은 곳에서 상품을 가져오는 코드

역할: 상품 검색, 상세 조회, 가격/재고 갱신

🌿 Git과 브랜치 — 왜 충돌이 안 나는가

Git은 코드의 모든 변경 이력을 기록하는 시스템입니다. Google Docs의 "버전 기록"과 비슷하지만, "내가 원할 때" 합치는 게 다릅니다. 이게 브랜치(Branch, 평행 세계) 개념입니다.

브랜치 개념도
┌── feature/tmon (A가 작업 중) ──── 커밋 ── 커밋 ── merge! │ ↓ main ────●────────────────────────────────────────────────────●────▶ │ ↑ └── feature/oliveyoung (B가 작업 중) ─ 커밋 ─ merge! 각자 다른 브랜치(=다른 파일)에서 작업하기 때문에, 충돌이 거의 없습니다.

핵심 용어: 커밋(Commit) = 저장, 푸시(Push) = GitHub에 업로드, 풀(Pull) = GitHub에서 다운로드, PR(Pull Request) = "코드 확인해주세요" 요청, 머지(Merge) = 브랜치를 합치기

개발 환경 세팅

한 번만 하면 되는 초기 세팅입니다. 순서대로 하나씩 따라하세요.

ℹ️
사전 설치 필요 없으면 팀장에게 알려주세요.
• Git — git-scm.com (설치 시 기본값 Next 연타)
• Python 3.12 — python.org ("Add Python to PATH" 반드시 체크!)
• uv — Python 설치 후 터미널에서: powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex" • Docker Desktop — docker.com (로컬 DB용)
• Claude Code — 팀장이 제공하는 설치 가이드로 설치
0

Git 전역 설정 (최초 1회)

Git을 처음 쓸 때 딱 한 번만 해주면 됩니다.

터미널 (Git Bash)
git config --global user.name "홍길동" git config --global user.email "hong@example.com" # GitHub 가입 이메일과 동일하게
1

GitHub에서 코드 가져오기 (클론)

프로젝트 코드를 내 컴퓨터에 복사합니다. 딱 한 번만 하면 됩니다.

터미널 (Git Bash)
cd ~/workspace git clone https://github.com/sbk0674-web/samba-wave.git cd samba-wave
2

백엔드 환경 만들기

파이썬 가상환경을 만들고, 필요한 패키지를 설치합니다.

터미널 (Git Bash)
cd backend uv venv source .venv/Scripts/activate # 터미널 맨 앞에 (.venv)가 보이면 성공 uv pip install -e . uv pip install -e .[dev]
3

로컬 DB + 환경 설정

터미널 (backend 폴더에서)
docker compose up -d # Docker Desktop이 실행 중이어야 함 cp .env.example .env # 수정할 것 없음, 그대로 사용

Antigravity에서 프로젝트 열기

1

Antigravity 실행 → 파일(File) → 폴더 열기(Open Folder) → samba-wave 폴더 선택

Git 작업하기

매일 반복하는 Git 루틴입니다.

🌅 아침 — 최신 코드 가져오기
💬 Claude Code에 말하기
"main에서 최신 코드 가져와서 내 브랜치에 merge해줘"

이걸 안 하면 옛날 코드 위에서 작업하게 됩니다. Claude Code가 checkout → pull → merge를 자동으로 해줍니다.

🆕 처음 시작할 때 — 브랜치 만들기
💬 Claude Code에 말하기
"쿠팡 플러그인 작업할 브랜치 만들어줘"
작업 종류 브랜치 이름 규칙 예시
판매처 feature/마켓이름-plugin feature/tmon-plugin
소싱처 feature/소싱처이름-sourcing feature/oliveyoung-sourcing
💾 커밋하기 (저장하기)

커밋 = 게임의 "세이브". 기능 하나 완성할 때마다 작게 자주 하세요.

💬 Claude Code에 말하기
/commit

Claude Code가 변경 파일 확인 → 내 파일만 선택 → 커밋 메시지 작성까지 자동으로 해줍니다.

🚨
git add . 절대 금지! 모든 파일을 추가해서 .env(비밀번호), 다른 사람 파일이 올라갈 수 있습니다. Claude Code를 쓰면 내 파일만 골라서 커밋해줍니다.
📝
커밋 메시지 규칙 [마켓] 쿠팡 플러그인 — transform 구현
[소싱] 올리브영 플러그인 — search 구현
[수정] 쿠팡 플러그인 — 인증 에러 처리 추가
☁️ 푸시 & PR (코드 올리기)

커밋 = 내 컴퓨터에 저장, 푸시 = GitHub에 업로드, PR = "코드 확인해주세요" 요청

💬 Claude Code에 말하기
"GitHub에 푸시해줘"
💬 Claude Code에 말하기
"PR 만들어줘"

Claude Code가 push + PR 생성까지 한번에 해줍니다. 팀장이 코드를 검토하고, 괜찮으면 main에 합쳐(merge)줍니다.

4

내가 작업할 파일

플러그인 파일은 이미 빈 틀이 만들어져 있습니다. 여러분이 할 일은 그 빈 틀 안에 실제 API 연동 코드를 채우는 것입니다.

🚫 절대 수정 금지 — 시스템 파일
파일역할
plugins/__init__.py플러그인 자동 탐색 코드
plugins/market_base.py판매처 기본 틀 (모든 판매처가 이걸 상속)
plugins/sourcing_base.py소싱처 기본 틀 (모든 소싱처가 이걸 상속)
collector/ 폴더 전체수집 엔진 (건드릴 일 없음)
🏪 판매처 담당이라면

아래 2개 파일을 작업합니다

파일역할
plugins/markets/내마켓.py 플러그인 본체
빈 틀 → 실제 연동 코드 채우기
proxy/내마켓.py HTTP 통신 담당
마켓 API와 실제 통신하는 코드
📦 소싱처 담당이라면

아래 2개 파일을 작업합니다

파일역할
plugins/sourcing/내소싱처.py 플러그인 본체
빈 틀 → 실제 연동 코드 채우기
proxy/내소싱처.py HTTP 통신 담당
소싱처에서 데이터 가져오는 코드
🏪 판매처 — Claude Code 프롬프트
💬 Claude Code에 말하기 — 예시
"롯데ON 판매처 플러그인에 실제 API 연동 코드 채워줘. 빈 틀이 markets/lotteon.py에 있어"
⚠️
판매처 점검항목 — 마켓에 정확히 들어가야 하는 것들
• 대표 이미지 — 상품 메인 썸네일
• 추가 이미지 — 상품 갤러리 이미지들
• 상세페이지 이미지 — 상품 설명 HTML 영역
• 고시 정보 — 소재, 제조국, 세탁방법 등 법적 필수 정보
• 태그 — 검색용 키워드 태그
• 옵션값 — 색상/사이즈 등 선택 옵션
• 상품명 — 마켓 규정에 맞는 상품명
• 카테고리 — 마켓 카테고리 코드 정확한 매핑

주문/CS 점검항목
• 주문 수집 — 주문번호, 수량, 결제금액, 배송지 등 모든 정보가 들어오는지
• 취소 승인 — 취소 요청 접수 및 승인 처리가 되는지
• CS 내용 — 고객 문의/클레임 내용이 정상 수신되는지
📦 소싱처 — Claude Code 프롬프트
💬 Claude Code에 말하기 — 예시
"롯데ON 소싱처 플러그인에 실제 API 연동 코드 채워줘. 빈 틀이 sourcing/lotteon.py에 있어"
⚠️
소싱 데이터 점검항목 — 이것들이 제대로 수집되어야 합니다
• 대표 이미지 — 상품 썸네일
• 추가 이미지 — 상품 갤러리 이미지들
• 상세페이지 이미지 — 상품 설명 영역의 이미지
• 판매가 — 원래 정가
• 할인가 — 실제 판매 가격 (최대 혜택가)
• 옵션 정보 — 색상/사이즈별 재고
• 정보고시 정보 — 소재, 제조국, 세탁방법 등
✅ 플러그인 테스트 점검
💬 Claude Code에 말하기
"서버 재시작해서 플러그인 등록 확인해줘"
⚠️
플러그인 숫자가 안 늘었다면?
  • 1파일 위치: plugins/markets/ 또는 plugins/sourcing/ 안에 있어야 함
  • 2market_type 또는 site_name 속성이 있는지 확인
  • 3MarketPlugin 또는 SourcingPlugin을 상속했는지 확인
  • 4로그에 로드 실패 메시지가 있는지 확인
💬 Claude Code에 말하기
"transform 함수 테스트해줘"
