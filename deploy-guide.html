1

Railway에 데이터베이스 만들기

1
railway.app 접속
GitHub 계정으로 로그인하세요.
2
New Project 클릭
대시보드 오른쪽 위에 보라색 버튼이 있어요.
3
Database → PostgreSQL 선택
메뉴에서 Database를 클릭하면 PostgreSQL이 보여요.
4
Variables 탭 클릭 → 연결 정보 메모
아래 6개 값을 메모장에 복사해두세요.
PGHOST = ???.railway.internal
PGPORT = 5432
PGUSER = postgres
PGPASSWORD = 어쩌구저쩌구긴문자열
PGDATABASE = railway
DATABASE_URL = postgresql://postgres:...@...railway.app:5432/railway
⚠️ 주의: DATABASE_URL에는 비밀번호가 포함되어 있어요. 절대 공개하지 마세요!
2

Railway에 백엔드 올리기

1
같은 프로젝트에서 + 버튼 클릭
왼쪽 패널에 + 아이콘이 있어요. 새 서비스를 추가합니다.
2
GitHub Repository → samba-wave 선택
GitHub에서 코드를 가져옵니다.
3
Settings 탭에서 3가지 설정
배포된 서비스를 클릭 → Settings 탭으로 이동
📁 Root Directory (소스 위치)
backend
🔨 Build Command (설치 명령어)
pip install -e .
🚀 Start Command (실행 명령어)
uvicorn backend.main:app --host 0.0.0.0 --port $PORT
4
Variables 탭에서 환경변수 입력
아래 내용을 하나씩 추가하세요. 1단계에서 메모한 DB 정보를 사용합니다.
# 환경
ENVIRONMENT = production

# DB 쓰기 (1단계에서 메모한 값)
WRITE_DB_USER = postgres
WRITE_DB_PASSWORD = (메모한 비밀번호)
WRITE_DB_HOST = (메모한 호스트)
WRITE_DB_PORT = (메모한 포트)
WRITE_DB_NAME = railway

# DB 읽기 (쓰기와 동일하게)
READ_DB_USER = (위와 동일)
READ_DB_PASSWORD = (위와 동일)
READ_DB_HOST = (위와 동일)
READ_DB_PORT = (위와 동일)
READ_DB_NAME = (위와 동일)

# 보안 키 (아무 긴 문자열)
JWT_SECRET_KEY = my-super-secret-key-12345678

# CORS (4단계 후 Vercel 주소로 변경)
CORS_EXTRA_ORIGINS = https://내앱.vercel.app
5
배포 완료 확인
자동으로 빌드가 시작됩니다. 초록색 체크가 나오면 성공!
Settings → Networking에서 Generate Domain 클릭하면 주소가 생깁니다.
예: https://samba-wave-production.up.railway.app
3

데이터베이스 테이블 만들기

DB를 만들었지만 아직 빈 상태예요. 테이블(상품, 주문, 정책 등)을 생성해야 합니다.

1
내 컴퓨터 터미널(명령 프롬프트)에서 실행
Railway의 DATABASE_URL을 사용해서 테이블을 만듭니다.
# 백엔드 폴더로 이동
cd backend

# Railway DB에 테이블 생성
set DATABASE_URL=postgresql://postgres:비밀번호@호스트:포트/railway
alembic upgrade head
✅ 성공하면: "Running upgrade ... -> head" 메시지가 나옵니다. 테이블 생성 완료!
4

Vercel에 프론트엔드 올리기

1
vercel.com 접속 → GitHub으로 로그인
2
Add New Project → samba-wave 레포 선택
3
설정 입력
📁 Root Directory
frontend
🔧 Framework Preset
Next.js (자동 감지됨)
4
Environment Variables 추가
2단계에서 받은 Railway 백엔드 주소를 입력합니다.
NEXT_PUBLIC_API_URL = https://samba-wave-production.up.railway.app
5
Deploy 클릭!
2~3분 후 배포 완료. Vercel이 주소를 줍니다.
예: https://samba-wave.vercel.app
5

CORS 연결하기 (마무리)

Vercel 주소를 Railway 백엔드에 알려줘야 서로 통신할 수 있어요.

1
Railway → 백엔드 서비스 → Variables 탭
2
CORS_EXTRA_ORIGINS 값을 Vercel 주소로 수정
CORS_EXTRA_ORIGINS = https://samba-wave.vercel.app
✅ 저장하면 자동으로 재배포됩니다. 1분 후 Vercel 주소로 접속해보세요!

🔧 문제가 생겼을 때

Q. 화면이 안 나와요
→ Vercel에서 NEXT_PUBLIC_API_URL이 올바른지 확인하세요.
Q. API 요청이 실패해요
→ Railway 백엔드 로그를 확인하세요. DB 연결 정보가 틀렸을 수 있어요.
Q. CORS 에러가 나요
→ CORS_EXTRA_ORIGINS에 Vercel 주소를 정확히 입력했는지 확인하세요. https:// 포함!
Q. 로컬 개발은 계속 되나요?
→ 네! 로컬은 그대로 유지됩니다. 배포는 복사본이에요. 안심하세요.
