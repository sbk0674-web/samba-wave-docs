🔢 2. 안전장치 — 왜 망가지지 않나요?

😱

안전장치 없을 때

직원이 실수로 코드 덮어쓰기
→ 실서비스 즉시 망가짐
→ 고객이 접속 불가
→ 복구하느라 밤샘 😭

😊

안전장치 있을 때

직원이 아무리 실수해도
→ 자기 공간에서만 영향
→ 사장님 승인 전까지 반영 안 됨
→ 실서비스는 항상 안전 👍

🛡️ 핵심 안전장치 3가지

① 브랜치 분리
각 직원은 자기만의 작업 공간(브랜치)에서 수정합니다.
다른 사람 작업에 절대 영향을 주지 않아요!
② PR (Pull Request)
"사장님, 제가 이렇게 수정했는데 확인해주세요" 라고 요청하는 것입니다.
사장님이 OK 해야만 반영됩니다.
③ main 보호
실서비스에 연결된 main 브랜치는 직접 수정이 불가능합니다.
반드시 PR → 승인을 거쳐야 합니다.

🔢 3. 직원 작업 순서 — 이것만 따라하세요!

📥 ① 최신 코드 받기
git checkout main → git pull
⬇
🌿 ② 내 작업 공간 만들기
git checkout -b feature/내이름-작업내용
⬇
✏️ ③ 코드 수정하기
Claude Code 등으로 마음껏 수정!
실수해도 내 공간이라 안전해요
⬇
💾 ④ 저장하고 올리기
git add . → git commit -m "설명" → git push
⬇
📮 ⑤ PR 생성 (확인 요청)
GitHub에서 "Pull Request" 버튼 클릭
→ 사장님한테 "PR 올렸어요~" 연락
⬇
✅ ⑥ 사장님 승인 → 자동 배포!
코드 확인 후 Merge 버튼 클릭
→ 실서비스에 자동 반영

🔢 4. 실수했을 때 — 전혀 걱정 마세요!

😰 이런 실수를 했어요 피해는? 해결 방법
코드를 다 날려버렸어요 내 브랜치만 브랜치 삭제 후 다시 만들면 끝!
main에 직접 push 했어요 피해 없음 보호 설정으로 자동 거부됨!
이상한 PR을 보냈어요 피해 없음 사장님이 승인 안 하면 반영 안 됨!
PC가 고장났어요 피해 없음 GitHub에 코드 다 있음. 새 PC에서 다시 받기!
다른 직원 파일을 수정했어요 충돌 가능 PR에서 충돌 표시됨 → 사장님이 정리!

🚫 이것만은 하지 마세요

① .env 파일을 git에 올리지 마세요 — 비밀번호가 들어있어요!
② 다른 직원의 브랜치를 건드리지 마세요 — 자기 브랜치에서만 작업!
