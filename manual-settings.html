1-1. 소싱처 쿠키 관리 (확장앱 연동)

무신사, KREAM 등 로그인이 필요한 소싱처는 확장앱을 통해 쿠키를 자동으로 수집합니다.

쿠키가 필요한 소싱처

  • 무신사 — 로그인 상태에서 최대혜택가(쿠폰 할인가) 수집 시 쿠키 필요
  • KREAM — 거래가 및 상품 정보 수집 시 로그인 쿠키 필요

쿠키 수집 방법

  1. 크롬 브라우저에서 해당 소싱처(무신사, KREAM 등)에 로그인합니다.
  2. 삼바웨이브 크롬 확장앱 아이콘을 클릭합니다.
  3. 확장앱이 자동으로 현재 로그인된 쿠키를 감지하여 서버에 전송합니다.
  4. 설정 페이지 소싱처 쿠키 탭에서 수집 상태를 확인할 수 있습니다.
TIP 쿠키는 만료될 수 있습니다. 수집 오류 발생 시 소싱처에서 재로그인 후 확장앱으로 쿠키를 다시 수집하세요.

1-2. 소싱처 계정 관리

확장앱과 연동되는 소싱처 계정 정보를 관리합니다. 계정별로 수집 범위와 쿠키를 독립적으로 관리할 수 있습니다.

계정 등록 방법

  1. 설정 > 소싱처 계정 탭으로 이동합니다.
  2. + 계정 추가 버튼을 클릭합니다.
  3. 소싱처를 선택하고 계정명을 입력합니다. (예: "무신사_계정1")
  4. 해당 계정으로 소싱처에 로그인 후 확장앱으로 쿠키를 연동합니다.
참고 동일 소싱처에 여러 계정을 등록하면 수집 시 계정을 선택하여 사용할 수 있습니다.

1-3. AI 설정

AI 기능(상품명 변경, 태그 생성, 이미지 변환) 사용에 필요한 API 키와 저장소를 설정합니다.

Gemini API Key (이미지 변환 / 상품명 / 태그)

Google Gemini API를 사용하여 AI 상품명 변경, AI 태그 생성, AI 모델 착용컷 이미지 생성을 수행합니다.

  1. Google AI Studio (aistudio.google.com)에서 API Key를 발급합니다.
  2. 설정 > AI 설정 > Gemini API Key 입력란에 붙여넣습니다.
  3. 저장을 클릭합니다.

Claude API Key (카테고리 / 텍스트 처리)

Anthropic Claude API를 사용하여 카테고리 자동 매핑, 상품 설명 텍스트 처리 등을 수행합니다.

  1. Anthropic Console (console.anthropic.com)에서 API Key를 발급합니다.
  2. 설정 > AI 설정 > Claude API Key 입력란에 붙여넣습니다.
  3. 저장을 클릭합니다.

Cloudflare R2 설정 (AI 이미지 저장소)

AI로 생성된 이미지(모델 착용컷 등)는 Cloudflare R2에 저장됩니다.

  • Account ID — Cloudflare 계정 ID
  • Access Key ID — R2 버킷 접근 키
  • Secret Access Key — R2 버킷 시크릿 키
  • Bucket Name — 이미지를 저장할 R2 버킷 이름
  • Public URL — R2 버킷의 공개 접근 URL
주의 API 키는 절대 타인과 공유하지 마세요. 유출 시 과금과 보안 사고가 발생할 수 있습니다.

2. API 키 발급 방법

각 마켓의 API 키를 발급받는 방법입니다.

스마트스토어

1. 네이버 커머스 API 센터 (https://apicenter.commerce.naver.com) 접속

2. 애플리케이션 등록 > 새 애플리케이션 생성

3. Client ID, Client Secret 발급

4. 삼바웨이브 설정에 입력

쿠팡

1. 쿠팡 윙 (wing.coupang.com) > 판매자 정보 > API 관리

2. Secret Key, Access Key 발급

3. Vendor ID 확인

4. 삼바웨이브 설정에 입력

11번가

1. 11번가 셀러오피스 > Open API 관리

2. API Key 발급 신청

3. 승인 후 API Key 확인

4. 삼바웨이브 설정에 입력

롯데ON

1. 롯데ON 셀러센터 > API 관리

2. API 인증 토큰 발급

3. 삼바웨이브 설정에 입력

SSG (신세계몰)

1. SSG 파트너센터 > API 관리

2. 인증 토큰 발급

3. 삼바웨이브 설정에 입력

* SSG는 계약 브랜드만 등록 가능합니다.

아마존 / 라쿠텐 / 바이마

해외 마켓은 각 마켓의 셀러 센터에서 API 키를 발급받습니다.

발급 절차가 마켓마다 다르므로 각 마켓의 공식 가이드를 참고하세요.

주의 API 키는 절대로 타인과 공유하지 마세요. 유출 시 판매자 계정이 위험해질 수 있습니다.

3. 인증 테스트

API 키를 입력한 후 연결이 정상적인지 테스트합니다.

  1. API 인증 정보를 입력하고 저장합니다.
  2. 인증 테스트 버튼을 클릭합니다.
  3. 연결 성공 시 녹색 "연결됨" 표시가 나타납니다.
  4. 연결 실패 시 에러 메시지를 확인하고 정보를 재확인합니다.

자주 발생하는 인증 오류

  • Invalid API Key — API 키가 잘못 입력됨. 복사-붙여넣기로 재입력
  • Expired Token — 인증 토큰이 만료됨. 마켓에서 재발급 필요
  • Permission Denied — API 권한 부족. 마켓에서 필요 권한 추가 설정
  • Network Error — 네트워크 연결 문제. 인터넷 연결 확인

4. 공통 설정

모든 마켓에 공통으로 적용되는 기본 설정을 관리합니다.

A/S 안내

마켓에 등록되는 상품의 A/S 안내 정보를 설정합니다.

  • A/S 전화번호
  • A/S 안내 문구
  • 교환/반품 주소

배송비 설정

  • 무료배송 기준 금액 — 이 금액 이상 구매 시 무료배송
  • 기본 배송비 — 무료배송 기준 미달 시 부과되는 배송비
  • 도서/산간 추가 배송비 — 도서산간 지역 추가 배송비

재고 설정

  • 기본 재고 수량 — 마켓에 표시할 기본 재고 수량
  • 품절 시 자동 판매중지 — 소싱처 품절 시 자동으로 판매 중지 여부

할인/혜택 설정

  • 포인트 적립률 — 고객에게 제공할 포인트 적립 비율
  • 리뷰 혜택 — 리뷰 작성 시 제공할 혜택 (포인트, 쿠폰 등)

5. 설정 저장

변경된 설정을 저장합니다.

  1. 설정 항목을 수정합니다.
  2. 페이지 하단의 설정 저장 버튼을 클릭합니다.
  3. 저장 성공 알림이 표시됩니다.
참고 설정 변경 후 반드시 저장 버튼을 클릭해야 합니다. 페이지를 이동하면 저장되지 않은 변경사항은 사라집니다.
