SECTION 03

검색 수집 흐름

"나이키" 키워드로 수집할 때 벌어지는 일을 순서대로 봐보자.

1

앱이 수집 요청

collector.js가 GET /api/kream/search?keyword=나이키&size=2 를 프록시에 요청함. 이 요청은 결과가 올 때까지 최대 90초 기다림.

collector.js → _collectKreamBulk()
2

프록시가 대기줄에 등록

프록시는 kreamSearchQueue 배열에 일감을 넣고 확장앱이 가져가길 기다림. 마치 식당 주문표 같은 것.

proxy-server.mjs → kreamSearchQueue
3

확장앱이 30초마다 폴링

확장앱은 알람으로 30초마다 /api/kream/search-queue를 확인함. 일감이 있으면 가져가서 처리 시작.

background.js → pollSearchOnce()
4

KREAM 검색 탭 오픈 + DOM 긁기

백그라운드 탭으로 kream.co.kr/search?keyword=나이키 열고, 3초 기다린 다음 상품 링크에서 상품 ID 목록을 파싱함. 50개 정도 나옴.

background.js → handleSearchJob()
5

결과 전달 → 개별 수집 루프 시작

검색 결과(상품 ID 목록)를 프록시에 POST. 프록시가 앱에 전달. 앱은 각 ID에 대해 단건 수집을 순서대로 호출함.

collector.js → _collectKreamSingle() 반복

SECTION 04

단건 수집 흐름

상품 하나를 수집할 때 어떻게 옵션(사이즈)까지 가져오는지 봐보자. 이게 핵심이야.

1

앱이 단건 수집 요청

GET /api/kream/products/41193 요청. 프록시는 kreamCollectQueue에 등록하고 역시 90초 대기.

collector.js → _collectKreamSingle()
2

확장앱이 폴링으로 일감 획득

이전 검색 처리 후 집중 폴링 모드로 전환됨 (3초 간격, 최대 20번). 수집 큐에서 일감을 가져옴.

background.js → pollCollectOnce()
3

상품 페이지 열기 + __NUXT__ 추출

상품 탭을 active로 만들고, JS 변수 __NUXT__에서 영문명·발매가·스타일코드 같은 정보를 꺼냄.

background.js → handleCollectJob() → __NUXT__
4

구매하기 버튼 클릭 → 사이즈 목록 수집

"구매하기" 버튼을 클릭해서 사이즈 선택 팝업을 열고, .select_item DOM 요소에서 사이즈 이름과 즉시 구매가를 읽음.

background.js → sizeOptions 수집
5

사이즈별 배송가격 수집 (별도 try/catch)

재고 있는 사이즈마다 클릭 → 배송 바텀시트에서 빠른배송·일반배송 가격 읽기. 이 부분만 별도 try/catch라 실패해도 사이즈 목록은 안 날아감.

background.js → 배송가격 루프 (별도 try/catch)
6

content script로 기본 정보 수집

kream-collect.js가 페이지에서 한글명, 브랜드, 이미지를 수집함. 사이즈 옵션은 background에서 이미 수집했으니 preCollectedOptions로 전달.

kream-collect.js → extractBasicInfo()
7

결과 조립 → 프록시 → 앱에 전달

모든 정보를 합쳐서 /api/kream/collect-result로 POST. 앱이 받아서 IndexedDB에 저장.

background.js → handleCollectJob() 완료

SECTION 05

폴링 엔진 — "할 일 있어요?" 기계

확장앱이 어떻게 타이밍을 맞추는지, 폴링 구조를 이해해야 버그를 잡을 수 있어.

// chrome.alarms로 30초마다 runPollCycle 실행
chrome.alarms.create('kreamPoll', { periodInMinutes: 0.5 })

async function runPollCycle() {
  const hadCollect = await pollCollectOnce()  // 단건 수집 큐 확인
  const hadSearch  = await pollSearchOnce()   // 검색 큐 확인
  if (hadCollect || hadSearch) {
    runFocusPoll()  // 일 있으면 → 집중 모드 진입 (3초 간격 × 20번)
  }
}

async function runFocusPoll() {
  let emptyCount = 0
  while (emptyCount < 20) {          // 최대 60초 집중
    const had = await pollCollectOnce() || await pollSearchOnce()
    had ? emptyCount = 0 : emptyCount++  // 일 있으면 리셋, 없으면 +1
    await wait(3000)
  }
  // 60초 동안 아무것도 없으면 → 다시 30초 alarm 대기 모드
}

💡 중딩 버전으로 설명하면

평소엔 30초마다 한 번씩 "일 있어요?" 물어봄. 일이 생기면 흥분해서 3초마다 물어보는 집중 모드로 전환. 60초 동안 아무것도 없으면 다시 졸면서 30초마다 물어봄.

상황 폴링 주기 최대 대기
평상시 (alarm 모드) 30초 무제한
일감 발견 후 (집중 모드) 3초 60초
요청 타임아웃 — 90초

SECTION 06

파일 지도 — 뭘 고치려면 어디 가야 해?

증상으로 파일 찾기.

이런 게 문제라면… 열어야 할 파일 찾아야 할 함수
수집 요청 자체가 안 감 collector.js _collectKreamBulk()
검색 탭이 안 열림 / DOM 파싱 실패 background.js handleSearchJob()
단건 수집 탭이 안 열림 background.js handleCollectJob()
사이즈 옵션 목록이 없음 background.js sizeOptions 수집 (구매하기 클릭 블록)
배송가격(fast/general)이 0임 background.js 배송가격 루프 (별도 try/catch 블록)
상품명이 이상한 텍스트로 잡힘 kream-collect.js extractBasicInfo() 필터 로직
이미지가 안 보임 / CORS 오류 background.js + proxy-server.mjs image-proxy API 경유 처리
수집 목표 수량이 이상함 collector.js autoCollectAndSave() collectTarget 계산
폴링이 너무 늦게 반응 background.js runFocusPoll() / setupAlarm()

SECTION 07

버그 기록부 — 같은 실수 두 번은 하지 말자

v2.0.0 ~ v2.0.2 동안 발생한 버그들. 왜 생겼고 어떻게 고쳤는지.

✅ [v2.0.0] 배송가격 수집 실패 시 사이즈 목록 통째로 날아감
배송시트 닫기 실패 등 에러 발생 → 외부 catch로 점프 → sizeOptions가 비어버림
→ 배송가격 루프만 별도 try/catch로 감쌈. 실패해도 sizeOptions는 유지됨.
✅ [v2.0.0] "51명이 보고 있습니다"가 상품명으로 저장됨
p.text-lookup 셀렉터가 너무 넓어서 조회수 텍스트까지 한글로 잡아버림
→ /\d+명/, /있어요$|있습니다$/ 등 필터 추가로 해결
✅ [v2.0.1] 검색 수집 시 옵션(사이즈) 정보 없음
검색 API 결과에 options: []만 있고 실제 사이즈 데이터가 없었음. 검색 결과를 그냥 저장했기 때문.
→ 검색으로 상품 ID 목록만 받고, 각 ID에 대해 단건 수집을 추가로 호출. 완전한 옵션 데이터 확보.
✅ [v2.0.1] 2개 요청했는데 3개 수집
Math.ceil(needed * 1.5) 공식 때문에 2 × 1.5 = 3으로 부풀려짐. KREAM은 중복이 없는데 여유분을 넣었던 것.
→ KREAM 실제 수집 모드(isKreamReal)는 collectTarget = needed 그대로 사용
✅ [v2.0.2] 로그가 수집 끝나고 한꺼번에 뜸
개별 수집 중 진행 로그가 없어서 "멈췄나?" 하고 기다릴 수가 없었음
→ 각 상품 수집 전 "수집 중..." 로그, 완료 후 "완료" 로그를 각각 추가
✅ [v2.0.2] KREAM 수집인데 [시뮬레이션] 태그 표시
isRealMode 판정에 무신사 URL 체크만 있었고 KREAM URL 체크가 빠져있었음
→ _isKreamUrl() 조건 추가. 이제 [실제수집] 태그 표시
✅ [v2.0.2] 품절 옵션에 가격이 남아있음
stock=0이거나 isSoldOut=true인 옵션도 kreamAsk 가격이 남아있었음
→ handleCollectJob에서 옵션 최종 조립 시 stock===0 || isSoldOut이면 price, kreamAsk를 0으로 처리

개발할 때 유용한 팁

다음에 또 버그 잡을 때 쓸 체크리스트.

확장앱 로그 보기

크롬 → chrome://extensions → SAMBA-WAVE → 서비스 워커 검사 클릭 → Console 탭. background.js 로그가 여기 뜸.

단건 수집 테스트

확장앱 팝업에서 "단일 상품 URL"로 직접 테스트. 검색 수집보다 훨씬 빠르고 결과 확인하기 좋음.

⚡

프록시 서버 확인

브라우저에서 localhost:3001 열어보면 API 목록 나옴. 안 뜨면 서버 꺼진 것. node proxy-server.mjs 로 실행.

⏱️

타임아웃 감안하기

확장앱 알람이 30초 주기라 최악의 경우 첫 반응까지 30초 걸릴 수 있음. 보통은 집중 폴링 덕분에 3~6초 안에 처리됨.

🔢

버전 업 필수!

extension 폴더 파일 수정하면 manifest.json의 version 번호 올리기. 안 올리면 크롬이 업데이트 안 함.
