Frontend → API Router → Service → Repository → DB  |  DDD 기반 domain/samba/ 구조

DB 세션 규칙 (필수)

수정/생성/삭제 → get_write_session_dependency()
조회 전용 → get_read_session_dependency()

핵심 페이지 - 상품 파이프라인

수집 → 가공 → 정책 → 매핑 → 전송

상품수집
/samba/collector
소싱처(무신사, KREAM, 다나와)에서 상품 수집. 필터 생성/관리, 실시간 수집 상태 확인, 수집 상품 미리보기.
Core 무한스크롤
상품관리
/samba/products
수집된 상품의 가격/마진 설정, AI 상품명 생성, 옵션 편집, 상세페이지 관리. 10만건 대응 무한스크롤.
Core AI
정책관리
/samba/policies
마켓별 가격 정책, 마진율, 배송비 규칙 설정. 상세페이지 HTML 템플릿, 상품명 생성 규칙 관리.
Core 템플릿
카테고리매핑
/samba/categories
소싱처 카테고리를 각 마켓 카테고리로 매핑. AI 자동 매핑 지원, 계층구조 트리 관리.
Core AI 매핑
상품전송
/samba/shipments
마켓별 상품 일괄 전송 및 삭제. 전송 상태 추적(pending/transmitting/completed), 에러 로그.
Core 대량처리
대시보드
/samba
주문 현황, 수집 통계, 전송 상태 요약. 오늘의 매출, 미처리 주문, 최근 활동 확인.

도메인 구조

backend/backend/domain/samba/ 내 각 도메인은 model / service / repository 구조

domain-structure
## 핵심 도메인 (상품 파이프라인) collector/ # 상품 수집 + 필터 + 갱신(refresher) product/ # 상품 CRUD + 무한스크롤 policy/ # 가격 정책 + 상세 템플릿 + 네이밍 규칙 category/ # 카테고리 매핑 (AI 벌크 매핑 포함) shipment/ # 마켓 전송 + 삭제 관리 plugins/ # 마켓(17) + 소싱(16) 플러그인 ## 운영 도메인 order/ # 주문 관리 + 클레임 감지 returns/ # 반품/교환/취소 처리 cs_inquiry/ # CS 문의 + 답변 템플릿 contact/ # SMS/카카오/이메일 발송 로그 analytics/ # 매출 통계 + KPI ## 인프라 도메인 account/ # 마켓 계정 인증 관리 user/ # 사용자 관리 tenant/ # 멀티테넌시 (SaaS 전환 중) cache/ # Redis + 인메모리 폴백 job/ # 백그라운드 작업 큐 image/ # 이미지 필터링 + 처리 proxy/ # 소싱처 프록시 (httpx 클라이언트) ai_sourcing/ # AI 소싱 분석 + 추천 forbidden/ # 금지어 + 글로벌 설정 warroom/ # 실시간 로그 + 대시보드 store_care/ # 스토어 건강도 + 정기 구매 probe/ # 소싱처 헬스체크

실행 명령어

개발 환경 실행에 필요한 주요 명령어

terminal
# 백엔드 실행 (backend 폴더에서) cd backend && .venv/Scripts/python.exe run.py # 프론트엔드 실행 (frontend 폴더에서) # 접속: http://localhost:3000/samba cd frontend && pnpm dev # DB 마이그레이션 적용 cd backend && .venv/Scripts/python.exe -m alembic upgrade head # 마이그레이션 파일 생성 (모델 변경 후) cd backend && .venv/Scripts/python.exe -m alembic revision --autogenerate -m "변경 내용 설명" # 마이그레이션 누락 검증 (model.py 수정 후 필수) cd backend && .venv/Scripts/python.exe scripts/check_migrations.py # 포맷 (push 전 필수) cd backend && .venv/Scripts/python.exe -m ruff format . # 린트 자동수정 (push 전 필수) cd backend && .venv/Scripts/python.exe -m ruff check --fix . # 타입 체크 cd backend && .venv/Scripts/python.exe -m mypy . # TypeScript 체크 cd frontend && npx tsc --noEmit # 프론트엔드 빌드 (CI/Vercel이 검증 — 로컬은 불필요) cd frontend && pnpm build
